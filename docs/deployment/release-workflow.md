# 发布流程指南

这份文档用于固定项目的发布前检查项，以及后续与 Codex 协作时的半自动发布流程。

## 1. 发布前必须检查的内容

每次准备发布时，至少确认以下事项：

1. 功能代码已经完成，并且本地核心流程可用
2. 重要页面没有明显的布局错乱、空白态异常或报错提示
3. 本次改动涉及的配置、域名、Supabase、地图能力没有被意外破坏
4. 需要同步更新的文档已经补齐
5. 本次发布的版本记录已经写入 `docs/product/changelog.md`
6. 如果这是一次正式对外发布，已经同步更新 `docs/product/release-notes.md`

补充规则：

- 只要当前一轮工作实际产生了代码、文档、配置或结构改动，就必须同步更新 `docs/product/changelog.md`
- 这条规则默认生效，不需要用户额外提醒

## 2. 发布时必须更新哪些文档

### 必更

- `docs/product/changelog.md`
  记录本次版本新增、优化、修复的核心变化；只要产生实际改动就更新

### 按需更新

- `docs/product/release-notes.md`
  只有“正式发布版本”才更新，用于整理更完整的发布说明
- `docs/product/requirements.md`
  只有需求范围、产品方向、阶段计划发生变化时才更新
- `docs/deployment/github-pages.md`
  只有部署方式、域名、环境地址变化时才更新
- `docs/backend/supabase.md`
  只有认证、数据库、RLS、配置流程变化时才更新

## 3. 版本文档的职责边界

- `changelog.md`
  用于持续追加版本记录，覆盖日常迭代、修复与阶段性更新
- `release-notes.md`
  用于正式发布摘要，内容比 changelog 更完整、更偏“对外说明”

规则：

- 不要把版本记录重新写回 `requirements.md`
- 只要这轮工作形成了实际改动，无论大小，都要更新 `changelog.md`
- 不要只更新 `release-notes.md` 而忘记 `changelog.md`
- 日常小修复可以只更新 `changelog.md`
- 正式上线版本应同时更新 `changelog.md` 和 `release-notes.md`

## 4. 半自动发布流程

以后你只要对我说：

```text
发布当前版本
```

我会按下面的固定流程执行：

1. 盘点本次代码与文档改动
2. 归纳本次版本主题和核心变化
3. 无论是否发布，都先更新 `docs/product/changelog.md`
4. 如果属于正式发布，再更新 `docs/product/release-notes.md`
5. 帮你检查是否需要同步更新部署、后端或需求文档
6. 按你的要求提交 commit
7. 如果你确认，再推送到远端并触发部署

这就是当前项目的“半自动发布流程”：

- 发布动作本身仍由你触发
- 版本文档由我按固定模板帮你维护
- GitHub Pages 仍然只负责部署，不负责自动写文档

## 5. 建议的版本更新格式

### 5.1 Changelog 模板

```md
## V18

- 新增：
- 优化：
- 修复：
```

说明：

- 如果某一类为空，可以不保留该小项，直接整理成 3 到 6 条高质量要点
- 每条尽量写“用户能感知到的结果”，少写纯实现细节

### 5.2 Release Notes 模板

```md
## V18

- 发布时间：2026-03-24
- 发布提交：`<commit>`
- 发布分支：`main`

### 本次版本重点

- 

### 主要更新说明

#### 1. 

- 

### 当前可用能力

- 

### 下一步建议

- 
```

## 6. 什么时候需要我一起更新 Release Notes

以下场景建议同步更新 `release-notes.md`：

- 正式上线新版本
- 有明显的新页面、新模块或新能力上线
- 域名、账号体系、后端能力发生阶段性变化
- 你准备对外展示、分享或归档版本成果

以下场景通常只更新 `changelog.md` 即可：

- 小型 UI 微调
- 单点修复
- 不影响整体产品叙事的技术性修补

## 7. 当前仓库的实际情况

- 当前仓库只有 GitHub Pages 部署工作流
- 没有“发布后自动生成 changelog / release notes”的脚本或 GitHub Action
- 因此版本文档目前是“人工触发 + Codex 协助整理”的模式
