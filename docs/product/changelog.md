# Changelog

## V70

- 新增独立“社区页”，并把原个人信息页中的好友中心总览、好友网络、私信对话与分享动态整体迁移过去。
- 顶部导航升级为 `首页 / 地点库 / 规划页 / 社区页 / 个人信息` 五页顺序，社区入口放在个人信息前面。
- 个人信息页回收为更纯粹的“我的”工作台，保留账号概况与旅行计划管理。
- 修复从其他页面切回地点库时卡片瀑布流未重新测量的问题，地点卡片现在会在地点库页激活后立即重新排布，不再出现整列堆叠后必须切换分类才能恢复的情况。

## V69

- 在个人信息页新增一版“好友中心”工作台，首轮接入推荐好友、好友申请、好友列表、私信会话和分享动态三栏结构，把加好友、聊路线、分享行程收拢到同一页完成。
- 新增账号维度的本地社交状态模型，支持发送好友申请、接受或忽略申请、打开私信、发送文本消息以及把当前绑定行程或当前规划草稿直接分享给好友。
- 补充好友能力的产品需求说明与 Supabase 后端预留设计，明确后续云端化时的表结构、RLS 边界和落地顺序。

## V68

- 将个人信息页下方的行程卡片改成瀑布式列表，并补充与规划页候选地点一致的悬浮波浪动画，浏览多条计划时的密度和层次感更强。
