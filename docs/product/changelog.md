# Changelog

## V32

- 地点库列表补充真实 masonry 布局：渲染后按卡片实际高度计算跨行数，并结合 `grid-auto-flow: dense` 让下方卡片上补空隙，避免各行顶部机械对齐。
- 新增地点库卡片在窗口尺寸变化后的重排逻辑，保证横向优先排列下仍能保持上下贴紧。

## V31

- 地点库卡片新增轻量动效：列表刷新时短促淡入上浮，悬停时卡片、标签与删除角标有细微抬升与阴影反馈。
- 增加 `prefers-reduced-motion` 兼容处理，在减少动态偏好下自动关闭地点卡动画。

## V30

- 地点库列表从按列优先的 `columns` 布局调整为按行优先的自适应网格：先横向铺满一行，再向下排列，保留卡片高度随内容自适应。
- 维持地点卡宽度基准不变，同时让卡片内容继续按自然高度展开，避免纵向优先排列带来的阅读跳跃。

## V29

- 地点库卡片布局从固定高度网格调整为瀑布流式排列：保持单卡宽度稳定，高度随内容自适应，列表按列连续铺排，提升信息利用率。
- 地点卡顶部改为“地名右侧显示分类标签”，减少竖向占用；底部改为“调整分类 / 省份自动识别”双栏同高布局，视觉更整齐。
- 保留原有品牌色系前提下，继续收敛地点卡信息密度，使地点库更接近收藏流式浏览体验。

## V28

- 修正地点库添加链路的误判去重逻辑：地点不再只按“名称 + 城市”粗略判断是否重复，而是改为更严格的地点身份键，避免同城不同点位被挡住。
- 地点搜索结果添加后继续保留当前结果列表，并只移除已加入项，方便连续收藏同一批搜索结果中的多个地点。
- 地点库添加后继续同步刷新页面摘要与收藏面板，避免新增内容被当前筛选状态或异步搜索刷新掩盖。

## V27

- 顶部导航升级为四页结构：`首页 / 地点库 / 规划页 / 个人信息页`，其中原“功能页”更名为“规划页”。
- 新增独立“地点库”页面，用于在正式规划前先收藏地点，并支持按 `玩 / 吃 / 住 / 其他` 分类及按省份筛选切换。

- 原“行程库”整合进新的“个人信息页”，作为“我的”页面首版基础，同时保留账号概况、地点概况和完整计划管理能力。

## V26

- 新增游客草稿登录后自动迁移：当游客模式下已有有效本地草稿时，用户登录或注册并进入会话后会自动创建对应云端计划并绑定为当前计划，避免体验链路在登录时中断。
- 打磨游客模式提示链路：未登录状态点击行程库或云端相关入口时，会回到账号中心并明确提示先登录，保留“先体验规划、后同步云端”的使用路径。
- 新增行程节奏诊断与时间冲突提醒：基于每天的出发时间、到达时间、路程和停留点数量，自动提示“时间冲突 / 结束偏晚 / 奔波偏多 / 节奏过满 / 节奏平衡”等状态，并在日卡片和右侧摘要中展示。
- 升级行程库基础管理：新增前端本地持久化的“置顶 / 最近打开”排序能力，并根据计划内容自动标记“待完善 / 进行中”，让列表层级更清晰。
- 收紧游客草稿迁云触发条件：当前本地已绑定云端计划时，不再重复创建新的云端副本，避免登录后误生成重复计划。
- 补齐行程库元数据清理与首页联动：云端计划刷新后会自动剔除失效的“置顶 / 最近打开”记录，首页“最近计划”也会优先展示最近打开和已置顶的计划。
