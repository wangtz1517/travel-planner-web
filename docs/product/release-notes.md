# Release Notes

## V71

- 发布时间：2026-03-27
- 发布分支：`main`

### 本次版本重点

- 地点库切换为“游客本地、登录云端”的双态存储模型，登录后会以当前账号的云端地点库为准。
- 新增地点库云端同步状态与最近同步时间展示，地点收藏是否已写入账号现在可以直接看见。
- 补齐 Supabase 地点库同步时间字段，并统一地点库“添加地点”模块说明文字的字级层次。

### 主要更新说明

#### 1. 地点库存储模型切换

- 登录后优先回读 `profiles.place_library_snapshot`，不再持续把浏览器本地缓存当成账号地点库主来源。
- 新增游客地点库与账号地点库的分离持久化逻辑，同一浏览器下只会执行一次性游客地点迁移，避免反复覆盖云端。
- 退出登录后会回到游客态地点库，账号数据和本地游客数据的边界更清晰。

#### 2. 地点库同步状态可视化

- 地点新增、删除和分类调整后，会继续回写账号云端快照，并同步更新 `profiles.place_library_synced_at`。
- 地点库摘要区域现在会显示本地 / 云端状态、同步中、同步失败和最近同步时间。
- 地点库“添加地点”模块下的说明文字已调回辅助说明层级，和页面其他说明文案保持一致。

#### 3. 后端与文档补齐

- 新增 `docs/backend/sql/003-profile-place-library-sync.sql`，用于补充地点库最近同步时间字段。
- `docs/backend/supabase.md`、`docs/product/requirements.md` 和 `docs/product/changelog.md` 已同步更新，记录当前地点库云端权威方案。

### 当前可用能力

- 账号登录后按账号维度读取、保存和同步地点库
- 游客地点库与登录地点库分离存储，并支持一次性迁移
- 页面内直接查看地点库当前同步状态与最近同步时间
- Supabase 地点库快照与同步时间字段已具备落库脚本

## V70

- 发布时间：2026-03-27
- 发布分支：`main`

### 本次版本重点

- 地点库升级为账号级云端快照，登录后可回读、合并并持续同步地点收藏。
- 新增独立“社区页”，把好友、私信和行程分享从个人信息页中拆出。
- 首页、地点库、规划页和个人页完成一轮信息架构与工作台体验收口。

### 主要更新说明

#### 1. 地点库云端化

- 为 `profiles` 补充 `place_library_snapshot`，支持账号级地点库云端持久化。
- 登录后会自动合并本地地点库与云端地点库，减少游客阶段收藏内容丢失。
- 地点库中的新增、编辑和删除会继续回写账号维度快照，不再依附单条旅行计划。

#### 2. 社区页独立成页

- 顶部导航升级为 `首页 / 地点库 / 规划页 / 社区页 / 个人信息` 五页结构。
- 原本放在个人信息页里的好友中心总览、好友网络、私信对话和分享动态已整体迁入社区页。
- 个人信息页回归“我的”工作台，只保留账号概况、地点收藏与计划管理。

#### 3. 工作台交互优化

- 首页补充更完整的首屏引导与阶段入口，当前计划和最近更新的聚合关系更清晰。
- 地点库与规划页的候选地点区继续收紧布局，提升浏览密度和拖拽前的筛选效率。
- 修复从其他页面切回地点库时瀑布流卡片整列堆叠的问题，页面激活后会自动重新排布。

### 当前可用能力

- 账号注册、登录与 Supabase 云端旅行计划保存
- 账号级地点库回读、合并与持续同步
- 独立社区页中的好友、私信和行程分享预览能力
- 首页概览、地点库、规划页、社区页、个人信息五页结构

### 下一步建议

- 补一次正式环境手动验收，重点确认地点库云端同步、社区页入口和页面切换后的布局稳定性
- 后续把社区数据从本地持久化继续推进到 Supabase 云端结构

## V24

- 发布时间：2026-03-25
- 发布提交：`193958e`
- 发布分支：`main`

### 本次版本重点

- 账号中心新增游客体验入口，未登录也能直接进入功能页体验路线规划与本地保存。
- 顶部登录状态改为可点击退出登录，已登录用户不用再去悬浮信息框里找退出入口。
- 账号中心的“注册 / 登录”切换恢复为单表单显示，避免两个表单同时出现。

### 主要更新说明

#### 1. 游客模式入口

- 未登录用户现在可以直接以游客身份进入功能页，先体验规划流程，再决定是否注册或登录。
- 游客模式仍保留本地保存体验，但云端保存、行程库等账号能力继续要求登录后使用。

#### 2. 登录交互收口

- 右上角状态徽标从纯展示改为可操作入口，登录后点击即可退出当前账号。
- 左上角品牌悬浮信息框改为纯信息展示，不再承载“进入功能页”和“退出登录”按钮。

#### 3. 账号中心表单修复

- 修复“注册 / 登录”切换时两个表单同时显示的问题。
- 为通用 `[hidden]` 状态补充强制隐藏规则，避免被表单布局样式覆盖。

### 当前可用能力

- 游客模式下可直接进入功能页体验路线规划与本地保存
- 已登录用户可继续使用账号、云端保存、行程库与归档管理
- 顶部导航、账号入口和品牌信息卡的登录交互更加统一

### 下一步建议

- 补一次浏览器侧手动验收，确认游客入口、登录切换和退出登录在正式环境下都顺畅
- 后续若继续扩展游客能力，可再评估是否加入游客提示或本地草稿引导


维护规则：

- 这份文档只记录正式发布版本，不记录每一次日常小改动
- 每次正式发布前，先更新 `docs/product/changelog.md`
- 当前仓库不会自动生成本文件，仍由人工触发、Codex 协助整理

## V23

- 发布时间：2026-03-25
- 发布分支：`main`
- 发布类型：工程结构重构版

### 本次版本重点

- 完成前端核心脚本的分层重构，当前工程不再依赖单个超大 `app.js` 承载全部逻辑。
- 清理原有同名函数重复定义问题，降低后续维护时误改未生效逻辑的风险。
- 将首页、行程库、功能页的主要渲染逻辑从总控脚本中拆出，工程结构更清晰。
- 补齐项目架构文档，方便后续继续迭代时快速理解文件关系和系统链路。

### 主要更新说明

#### 1. 前端结构重构

- 将原本集中在 `assets/js/app.js` 的业务逻辑拆分为六层脚本：
  - 状态层
  - 认证层
  - 规划层
  - 地图层
  - 足迹层
  - 页面渲染层
- `assets/js/app.js` 现在只负责启动流程和事件绑定，职责明显收敛。
- `index.html` 已改为按新的分层顺序加载脚本。

#### 2. 维护风险收敛

- 清理了原先依赖“后定义覆盖前定义”的重复函数问题。
- 当前每个核心能力只保留一份最终生效实现，更适合继续维护和迭代。
- 首页、行程库、功能页的渲染职责已集中到渲染层，页面之间的耦合有所降低。

#### 3. 文档与认知成本优化

- 新增项目架构文档，补充文件关系图和系统关系图。
- 更新开发文档入口，明确新的前端脚本结构。
- 同步更新版本变更记录，确保工程改造与文档描述保持一致。

### 当前可用能力

- 多日旅行规划与地点编排
- Supabase 账号注册、登录和云端计划保存
- 行程库管理、归档、复制、删除
- 首页旅行概览与足迹地图展示
- 正式域名访问：`https://www.gopace.cn/`

### 下一步建议

- 把当前全局共享变量继续收敛为统一状态容器
- 逐步从经典脚本结构过渡到 ES Module
- 为 Supabase、地图和足迹识别补充更稳定的测试与调试入口

## V14

- 发布时间：2026-03-24
- 发布提交：`51ed313`
- 发布分支：`main`

- 完成账号体系第一阶段接入，支持注册、登录、退出和基础个人资料读取。

- 接入 Supabase Auth，支持邮箱注册和登录。
