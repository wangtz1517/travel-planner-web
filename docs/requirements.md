# 旅行规划网页需求文档

## 1. 项目目标

做一个供个人旅行规划使用的网页工具，用于在出发前整理多日行程、地点顺序、停留时间、交通方式，并在地图上可视化展示每日路线，最后导出 PDF 方便分享。

参考产品：圆周轨迹。

## 2. 目标用户

- 当前主要用户：你自己
- 典型使用场景：
  - 已经确定了出游人数
  - 已经确定了出游日期
  - 已经初步收集到本次旅行可能会去的地点
  - 需要按天规划顺序、时间和交通方式
  - 需要地图可视化结果与可分享的 PDF

## 3. 核心流程

1. 输入本次旅行基础信息
2. 搜索并建立本次旅行的候选地点库
3. 系统根据日期自动生成每日行程容器
4. 将候选地点拖入某一天，并按顺序排列
5. 通过拖动调整同一天内地点顺序
6. 为每个地点设置类型和停留时间
7. 为相邻两个地点之间设置出行方式
8. 系统计算每日总停留时间、路程和预估出行时间
9. 地图展示该日全部地点与路线
10. 导出整趟旅行的 PDF

## 4. 基础信息

当前版本仅保留以下字段：

- 旅行名称
- 出行人数
- 开始日期
- 结束日期

说明：

- 日期范围用于自动生成 Day 1、Day 2、Day 3 等每日区域
- 地图 Key 不在表单中输入，改为单独配置文件管理

## 5. 地点库

每个地点保留以下信息：

- 地点名称
- 完整地址
- 所属城市
- 经度
- 纬度
- POI ID（如可获取）
- 备注

录入方式：

- 用户输入关键词或地址
- 直接在搜索框下拉结果中查看最多 5 条地点结果
- 每条结果右侧提供“添加”按钮，可直接加入地点库
- 城市、地址、经纬度等信息由搜索结果自动填充

地点库特点：

- 一次旅行可录入多个候选地点
- 候选地点可重复加入不同行程
- 同一个地点拖入行程后，生成行程项实例，不影响原始地点库

## 6. 每日行程项设置

每个被拖入某一天的地点块，至少需要配置以下信息：

- 关联地点
- 类型
- 停留时长
- 开始时间（可选，后续扩展）
- 备注

### 6.1 地点类型建议分类

首版采用以下分类：

- 起点
- 终点
- 景点
- 途径点
- 饭店
- 酒店
- 交通枢纽
- 购物
- 休闲
- 备用

### 6.2 行程项排序

- 同一天内，地点块支持拖拽排序
- 支持从候选地点库拖入每日行程
- 后续可扩展为跨天拖拽移动
- 顺序决定地图编号和路线连接顺序

## 7. 出行方式设计

两个相邻地点之间可设置一个出行方式。

首版建议支持：

- 步行
- 打车
- 自驾
- 公交 / 地铁
- 骑行
- 火车
- 飞机
- 轮船
- 其他

## 8. 自动计算逻辑

系统需要根据地点顺序和交通方式，计算：

- 每段路程距离
- 每段预计出行时间
- 每个地点停留时长
- 每日总行程时间
- 每日总路程

首版规则：

- 如果高德 API 可用，优先使用高德路线服务获取步行、驾车、公交、骑行等时间和路程
- 对于火车、飞机、轮船等方式，首版采用估算逻辑
- 如果 API 不可用，则所有方式先按估算规则计算，保证网页仍可使用

## 9. 地图展示需求

地图区域需要支持：

- 展示当日全部地点标记
- 按顺序显示编号
- 区分起点、终点和中间点
- 根据顺序连接路线
- 不同出行方式使用不同颜色
- 估算路径可用虚线显示
- 提供路线图例
- 自动缩放到当日路线范围

## 10. PDF 导出需求

导出内容包括：

- 旅行基础信息
- 地点库
- 每日详细行程
- 每日统计
- 地图视图
- 当前查看日的路线分段摘要

实现策略：

- 首版采用浏览器打印能力导出 PDF
- 提供针对打印场景的专门样式
- 打印时改为纵向分块排版，避免屏幕三栏布局影响阅读

## 11. 数据保存

首版支持：

- 浏览器本地自动保存
- 手动清空重置

## 12. 当前版本功能范围

当前版本实现以下功能：

- 新建旅行基础信息
- 根据日期自动生成多日区域
- 搜索框下拉展示地点结果并直接加入地点库
- 拖动地点到每日行程
- 拖动调整行程顺序
- 设置地点类型
- 设置停留时间
- 设置相邻段出行方式
- 自动估算距离和时间
- 地图展示当日点位和路线
- 本地保存
- PDF 导出

## 13. 高德地图接入方案

### 13.1 配置方式

高德 Key 不再在页面表单中输入，改为通过本地配置文件管理。

建议使用：

- `config.js`：本地实际配置，不提交公开环境时可自行保留
- `config.example.js`：示例配置

配置内容建议包括：

- Web 端 JS API Key
- 安全密钥 `securityJsCode`
- 默认搜索城市

### 13.2 申请方式

需要去高德开放平台申请。

基本流程：

1. 注册并登录高德开放平台开发者账号
2. 在控制台创建应用
3. 为应用添加 Web 端 JS API 的 Key
4. 获取 Key 和安全密钥
5. 将 Key 和安全密钥填入本地 `config.js`

### 13.3 使用建议

- 开发阶段可使用明文方式配置 `securityJsCode`
- 正式生产环境更建议走代理服务器转发方式
- 本地静态页面首版先采用配置文件方式，方便你自己使用

## 14. 版本更新记录

### V1

- 初版需求整理完成
- 支持基础信息、地点库、按天规划、路线估算、地图展示、PDF 导出

### V2

本轮新增与调整：

- 界面改为横向布局：基础信息在上方横条，候选地点库在左，多日行程在中，地图预览在右
- 整体布局改为紧凑模式，尽量减少整页上下滚动
- 基础信息精简为：名称、人数、时间
- 高德 Key 从页面表单中移除，改为单独配置文件方式
- 候选地点库改为搜索式录入，不再手填城市、地址、经纬度
- 地点加入行程后，支持拖拽排序，替代按钮上下移动
- 增加高德申请和配置说明，便于后续正式接入

### V3

本轮新增与调整：

- 搜索框支持联想搜索，输入时自动拉取候选项
- 支持键盘交互：方向键选择候选项，回车直接选中并搜索，Esc 关闭候选项
- 搜索结果列表展示更完整的城市、区县、类型和坐标信息
- 地图增加路线图例、起点终点区分、估算虚线和更清晰的标注样式
- 右侧摘要增加“当前查看日”的分段交通说明
- 打印导出样式重新整理，导出 PDF 时转为更适合阅读和分享的纵向布局

### V4

本轮新增与调整：

- 删除页面中关于高德配置方式的提示文案，界面更干净
- 地图预览改为固定地图挂载层和独立空状态层，避免切换日期或刷新后地图不显示
- 地图容器增加显式高度与 `resize` 处理，提升高德地图在当前布局中的显示稳定性
- 同步保留现有联想搜索、路线图例、分段摘要和 PDF 导出能力

### V5

本轮新增与调整：

- 删除地图预览下方的状态说明文案
- 中部“多日行程规划”和右侧“地图预览”调整为接近 1:1 宽度，提升地图可视面积
- 候选地点联想改为搜索框下拉列表形式，每次最多显示 5 条，离开输入区后自动消失
- 删除候选地点库标题下方的辅助解释文案，界面进一步收敛

### V6

本轮新增与调整：

- 删除“找到 N 条相关地点”和中间搜索结果区域
- 搜索流程改为单层下拉结果，搜索框下方直接显示地点列表
- 每条地点结果右侧增加“添加”按钮，点击后可直接加入地点库
- 搜索交互减少一次中间过渡，候选地点录入更直接

### V7

本轮新增与调整：

- 左侧候选地点库进一步简化为仅保留搜索框，下拉列表中直接显示地点候选与“添加”按钮
- 删除地点备注输入框与搜索按钮，搜索后可直接加入下方地点库，减少一次中间操作
- 地点库中的每个地点卡片新增独立备注输入框，便于对单个地点补充提示信息
- 左侧地点卡片整体压缩高度，候选地点库布局更紧凑
- 多日行程中的出行方式改为显示在两个地点之间的独立区段卡片中，顺序关系更直观
- 当一天内存在两个及以上地点时，自动显示相邻区段的出行方式选择框与路程时间估算
- 行程地点卡片与备注输入整体收紧，减少无效留白，提升单屏可见信息量

### V8

本轮新增与调整：

- 多日行程时间逻辑改为以起点出发时间为基准自动推导，后续地点自动计算到达时间与离开时间
- 起点仅输入出发时间，中间地点显示自动到达时间并只设置停留时间，终点自动显示到达时间
- 多日行程增加按天切换按钮，编辑时可直接切换 Day 视图，不再需要长距离上下滚动
- 左侧地点库卡片移除备注输入，卡片高度固定并压缩，地点数量变化时卡片尺寸不再跳动
- 顶部右上角操作按钮恢复垂直居中，与基础信息栏保持更稳定的对齐
- 地图标记改为自定义数字圆点，修复序号乱码问题
- 地图路线优先尝试调用高德步行、驾车、公交、骑行服务绘制真实路径，跨城或不支持场景再回退为直连线
